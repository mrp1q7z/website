第３章　機能一覧

バグ管理

機能 説明
新規バグ 新たにバグ票を発行します。
バグ一覧 登録されているバグの一覧を表示します。
バグ検索 バグの状態、発生プログラム等でバグの検索を行います。
バグ詳細 バグの詳細な内容を表示し、登録内容の変更を行います。
添付資料 バグに添付された資料（画像ファイル等）を追加・削除します。
バグ曲線 発生したバグの累積件数をグラフで表示します。
バグが減っているのか増えているのかを判断するために使用します。
工数集計 バグの対処に要した時間を集計します。

リリース管理

機能 説明
新規リリース プログラムのリリースをバージョン番号を付けて記録します。
リリースログ いつ何をリリースしたのかの一覧を表示します。
バージョンAとバージョンBの相違点（対処したバグの一覧）を調べることができます。

設定

機能 説明
ユーザーメンテ ユーザーの追加、削除、変更を行います。
プログラムメンテ プログラムの追加、削除、変更を行います。
プロジェクトメンテ プロジェクトの追加、削除、変更を行います。
権限設定 各画面ごとの権限（全て不可／参照のみ／全て可）を設定します。
オプション バグ一覧に表示する行数、バグ詳細を別ウインドウにするかの設定等を行います。

その他メニュー

機能 説明
ログアウト 初期画面に戻ります。別のプロジェクトを開くときは一旦ログアウトします。
XML取込 XML形式のバグ情報を読込み、バグまねに登録します。
ヘルプ このヘルプファイルを表示します。
サポート 作成者のホームページを表示します。バグ・要望等があれば、どんどんメールください。

バグの重要度一覧

重要度 説明
要望 機能強化の要望。
小欠陥 スペルミス等の機能や動作にはあまり影響を与えない重要でないもの。
中欠陥 欠陥があるが、簡単な代替手段があるもの。
大欠陥 機能が動かないなどの大きな欠陥があるもの。
重大 クラッシュやデータの破損、メモリリークなどの重大なもの。
極めて重大 開発やテストができなくなるくらいに極めて重大なもの。

バグの状態一覧

状態 説明
New 新しく報告されたバグ。
担当就任 バグを処理する担当者が決まった状態。
再開 一旦はバグが解決されたと思われたが、何らかの理由でバグが再発した状態。
解決済み バグが解決されて確認を待っている状態。
確認済み バグが解決されたことが確認された状態。
終結 バグを解決した製品が無事に出荷することができて、このバグが無効になった状態。

バグのライフサイクル


発見者 発見したバグを新規登録する。
また、変更要望等があった場合も新規にバグ票を登録する。
バグはバグ受付者に回される。
バグ受付者 バグ内容を判断し、バグの対処者・対処期限等を設定する。
ここでバグでない（仕様どおり）と判断されバグか完了する場合もある。
バグであった場合、バグ対処者に回される。
バグ対処者 バグの対処を行い、対処内容を記入する。
バグはバグ確認者に回される。
確認者 対処済みバグの確認を行い、確認結果を記入する。
確認がＯＫの場合、バグは完了する。
確認がＮＧの場合、バグ受付者を経由し再度バグ対処者に回される。
