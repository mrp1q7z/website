XML取込


概要 XML形式のバグ情報をバグまねに取込みます。エクセル形式のバグ情報を取込む場合は、以下を参照してください。
画面遷移 その他メニューの［XML取込］で表示されます。

項目説明
XMLファイル XMLファイル名を入力します。
取込先プロジェクト 取込先プロジェクトを選択します。
・既存のプロジェクトに追加するには、既存のプロジェクトを選択し、上書きモードのチェックをOFFにします。
・既存のプロジェクトをXMLの内容に置き換えるには、既存のプロジェクトを選択し、上書きモードをONにします。
・新規プロジェクトとして取込むには、取込先プロジェクトを未選択にします。上書きモードはどちらでも同じです。
上書きモード チェックをOFFにしますと、取込先プロジェクトに追加されます。
チェックをONにしますと、バグ情報・リリース情報等はすべて削除され、XMLの内容に置き換えられます。

操作説明
参照 XMLファイルを選択するファイル選択ダイアログが表示されます。
実行 XML取込を開始します。

●バグIDは振りなおされます。
例：既存のプロジェクトに１０件のバグ情報があった場合、バグID11〜として取り込まれます。

●新規プロジェクトとして取り込んだ場合、”取込_新しいプロジェクト”という名前のプロジェクトが作成されます。
取り込み後に適切な名前のプロジェクトに変えてください。また、プロジェクトに参加するユーザーも設定されていません
ので、プロジェクトメンテのユーザーの編集で参加するユーザーを設定してください。

●XMLデータで出現したプログラム、ユーザーがマスタになければ自動で登録されます。

●ライセンス未登録の場合は、１００件までしか取り込むことはできません。

エクセル形式のバグ情報を取り込むには
エクセル形式のバグ情報を直接取込むことはできません。一旦XMLに変換してください。
エクセル→XML変換は、有償・無償のツールがたくさんありますのでそちらで変換してください。

XMLタグ一覧

タグ 説明
ROWSET 行の集合。バグ情報全体をこのタグでくくります。
ROW バグ情報１件ごとに、このタグでくくります。
bug_id バグIDを記述します。取り込み時に振りなおされますが、エラー個所の特定時に必要です。
例）バグID99のプログラム名が長すぎる場合、以下のようなエラーが出力されます。
→「bug_id99:プログラム名の文字数がオーバーしています。(32byteを超えました)」
program_name プログラム名を記述します。同じプログラムは同じ名前にしてください。
"プログラムa"と"プログラムA"と記述しますと、プログラムマスタには2件登録されます。
program_memo プログラム説明を記述します。
priority_name 以下のいずれかで、重要度を記述します。
要望/小欠陥/中欠陥/大欠陥/重大/極めて重大
carer 対処者を記述します。
status_name 以下のいずれかで、状態を記述します。
New/担当就任/再開/解決済み/確認済み/終結
deadline 対処期限を"YYYY/MM/DD"で記述します。
summary 要約を記述します。
find_date 発生日時を"YYYY/MM/DD HH:MM:SS"で記述します。
version バージョンを記述します。
component コンポーネントを記述します。
detail 詳細を記述します。
finder 発見者を記述します。
relation_bug 関連バグのバグIDを記述します。
cause 原因を記述します。
care_detail 対処内容を記述します。
care_clocking 対処に要した時間を記述します。単位は時間です。30分の場合は0.5
care_date 対処日時を"YYYY/MM/DD HH:MM:SS"で記述します。
ok_flg 確認がOKかどうかを記述します。OK又はNGのいずれか。（半角の大文字で記述してください）
checker 確認者を記述します。
check_date 確認日時を"YYYY/MM/DD HH:MM:SS"で記述します。
comment 確認時コメントを記述します。
